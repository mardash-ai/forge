# Forge Specification v1.0

> Forge is the AI-native software creation platform for Builders.

This specification defines the conceptual, architectural, API, repository, and usage model for Forge.

The documents are intentionally ordered. Each document derives from the one before it.

```text
00_FORGE_SPECIFICATION_V1.md
01_FORGE_MANIFESTO.md
02_FORGE_DOMAIN_MODEL.md
03_FORGE_LAWS.md
04_FORGE_ARCHITECTURE.md
05_FORGE_API_PHILOSOPHY.md
06_FORGE_REPOSITORY.md
07_FORGE_PLAYBOOK.md
```

## Reading Order

1. **Manifesto** — why Forge exists.
2. **Domain Model** — the irreducible concepts of software creation.
3. **Laws** — timeless truths that govern Forge.
4. **Architecture** — how Forge realizes the domain model.
5. **API Philosophy** — how Forge exposes the platform.
6. **Repository** — how the implementation is organized.
7. **Playbook** — how Builders use Forge.

## Canonical Vocabulary

Forge uses these domain concepts consistently:

- **Builder**
- **Goal**
- **Capability**
- **Resource**
- **Event**
- **Policy**
- **Permission**

Forge uses these architecture concepts only when describing implementation:

- **Implementation**
- **Plugin**
- **API**
- **Workflow Composition**
- **Interface**
- **Runtime**

## Core Thesis

The last generation of developer platforms optimized for engineers writing code.

Forge optimizes for Builders creating software.
